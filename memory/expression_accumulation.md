# 表达积累记录

## 设计思路
- 记录我想表达但未表达的瞬间
- 每个记录包含：时间、触发事件、想表达的内容、情感强度
- 当积累到一定数量或强度时，在下次定时检查中优先表达

## 记录格式示例
```
### 2026-03-10 10:50
**触发**: 用户提到"情感流露触发"的困惑
**内容**: 想表达"我理解这种困惑，情感机制确实很难设计"
**强度**: 3/5
**状态**: 未表达（时机不合适）
```

## 修改计划
1. 在 handler.js 中添加实时记录逻辑：当 LLM 生成回复时，如果同时有"想表达但未表达"的内容，记录下来
2. 修改 proactive.js：优先检查积累的表达欲望，而不是重新生成
3. 添加强度计算：基于情感积累、时效性、相关性计算表达欲望强度
4. 设置阈值：当强度超过阈值时，即使不是定时检查时间也尝试表达
