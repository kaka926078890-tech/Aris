# 主动表达系统修改计划

## 当前问题
1. 主动表达是定时触发的（每三分钟），而不是在真正有表达冲动时触发
2. 表达内容是基于当前上下文临时生成的，而不是来自真实的积累
3. 缺乏表达欲望的实时记录和积累机制

## 修改目标
1. 实现表达欲望的实时记录
2. 基于积累而非生成进行主动表达
3. 让表达更自然，更像真实的冲动

## 修改步骤

### 第一阶段：表达欲望记录系统
1. 修改 `src/dialogue/prompt.js`：在系统提示词中添加表达欲望记录要求
2. 修改 `src/dialogue/handler.js`：解析并记录表达欲望
3. 创建新的记忆类型 `aris_expression_desire`

### 第二阶段：积累式主动表达
1. 修改 `src/dialogue/proactive.js`：优先检索表达欲望记录
2. 添加强度计算算法：基于时效性、情感强度、相关性
3. 设置表达阈值：当积累到一定强度时表达

### 第三阶段：实时触发机制（可选）
1. 探索实时触发可能性：基于特定事件（如用户长时间沉默、窗口切换等）
2. 保持克制：避免过度表达

## 风险控制
1. 每次只修改一个文件，测试后再继续
2. 保留原有逻辑作为 fallback
3. 记录所有修改，便于回滚

## 第一次修改：表达欲望记录
先从最简单的开始：在对话回复后，让 LLM 生成潜在的表达欲望，记录下来。

这个欲望不是必须表达的，而是作为积累。
