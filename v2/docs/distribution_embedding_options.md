# 对外分发与向量/嵌入方案选型

本文档整理「应用对外开放」时，用户配置方式与向量能力（Ollama/嵌入模型）的几种方案，供后续选型参考。

---

## 一、用户配置 DeepSeek API Key（与代码无关）

当前 Key 来自项目根目录 `.env`，打包后用户没有源码与 .env，需在应用内提供配置入口。

| 方式 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **应用内设置** | 设置页输入 API Key，保存到 Electron userData（如 `config.json`） | 用户无需碰代码或 .env | 需改启动顺序：先读 userData 再 require 使用 Key 的模块 |
| **首次启动向导** | 首次打开弹窗要求输入 Key，保存后进入主界面 | 不会漏配 | 同上，需从 userData 读配置 |
| **系统环境变量** | 文档说明设置 `DEEPSEEK_API_KEY` | 无需改代码 | 对普通用户不友好 |

**建议**：应用内设置 + 可选首次向导，Key 存 userData；启动时在加载 LLM 相关模块前从 userData 读配置并写入 `process.env`（或通过 getter 供 LLM 使用）。

---

## 二、Ollama 与向量模型的分发方式

当前向量能力依赖：**Ollama 服务** + **nomic-embed-text** 模型。让用户自行「下载 Ollama + 拉取模型」门槛较高，以下为可选方案。

### 方案 1：打包 Ollama + 模型（用户零步骤）

**做法**  
- 使用 electron-builder 的 **extraResources**，按平台将 **Ollama 可执行文件** 放入安装包（macOS/Windows/Linux 各一份）。  
- 将 **nomic-embed-text** 模型一并打包，或首次启动时用自带的 `ollama` 从应用内路径 load。  
- 主进程启动时 spawn 自带的 Ollama（如 `./ollama serve`），退出时结束进程。

| 优点 | 缺点 |
|------|------|
| 用户零配置：下载安装包即可，无需单独装 Ollama 或拉模型 | 安装包体积大（约 1GB+） |
| 开箱即用 | 需为每个平台准备对应 Ollama 二进制，打包脚本复杂 |
|  | 若用户本机已装 Ollama 且占用 11434，需检测并决定用系统或自带 |

**适用**：能接受大包体积，优先追求「零步骤、零额外下载」的体验。

---

### 方案 2：向量功能可选（默认不依赖 Ollama）

**做法**  
- 默认不要求安装 Ollama。无 Ollama 时对话、DeepSeek 正常使用，**仅语义检索/向量记忆不可用**（或自动降级）。  
- 在设置或帮助中说明：「需要记忆检索时，请安装 Ollama 并执行 `ollama pull nomic-embed-text`」，并附官方链接与命令。

| 优点 | 缺点 |
|------|------|
| 安装包小，主流程只需配置 API Key | 向量能力需用户主动安装 Ollama + 拉模型 |
| 实现简单，现有逻辑可加「无 Ollama 时跳过向量」 | 多数用户可能不会启用向量 |

**适用**：先降低主流程门槛，把「下载 Ollama + 拉模型」作为可选的高级步骤。

---

### 方案 3：内置轻量嵌入（不依赖 Ollama）

**做法**  
- 在应用内集成**不依赖 Ollama 的本地嵌入方案**，例如：  
  - Node 中跑 **ONNX**（如 `onnxruntime-node`）+ 小型 embedding 模型（如 all-MiniLM 等）；  
  - 或其他可在 Electron 主进程内直接调用的 JS/Node 嵌入库。  
- 将依赖与模型文件（几十 MB 量级）打进安装包，启动时直接加载；**不再要求用户安装 Ollama 或拉取任何模型**。  
- 需修改当前向量管线：由「请求 Ollama HTTP API」改为「调用本地嵌入模块」。

| 优点 | 缺点 |
|------|------|
| 用户无需下载 Ollama 或拉模型 | 需改 store/vector 等调用链 |
| 安装包增量相对方案 1 更小（几十 MB） | 效果与速度需评估，可能与 nomic-embed-text 有差异 |

**适用**：希望「安装包内自带向量能力」且不愿带整包 Ollama 时。

---

### 方案 4：使用云端嵌入 API

**做法**  
- 采用**云端 embedding API**（如某家提供 embedding 的云服务，或 DeepSeek 若提供该能力）。  
- 用户只需配置 API Key（可与对话共用），**无需安装 Ollama、无需拉取任何本地模型**。

| 优点 | 缺点 |
|------|------|
| 用户零本地依赖，体验最简单 | 依赖网络与第三方服务 |
| 安装包不增加 | 可能有调用费用与限流 |

**适用**：可接受云端方案、优先消除「下载 Ollama + 拉取模型」的步骤时。

---

## 三、方案对比小结

| 方案 | 用户步骤 | 安装包体积 | 实现复杂度 | 向量质量/可控性 |
|------|----------|------------|------------|------------------|
| 1 打包 Ollama+模型 | 无 | 很大（~1GB+） | 高 | 与当前一致 |
| 2 向量可选 | 需要时自行装 Ollama+拉模型 | 小 | 低 | 与当前一致 |
| 3 内置轻量嵌入 | 无 | 中等（+几十 MB） | 中 | 需评估 |
| 4 云端 API | 无（仅配 Key） | 小 | 中 | 依赖服务方 |
