# Aris 后续演进方向（备忘）

本文档记录**产品仍可继续升级**的几条主线，便于日后查阅与排期；与「当前是否已做完某一版」无关——软件会持续演进。

> 与具体迭代任务、优先级对照时，可结合 [ux_optimization_implementation_plan.md](ux_optimization_implementation_plan.md)、[plan.md](plan.md) 等。

---

## 1. 产品 / 体验

- 新能力、新工作流（例如会话组织、搜索、快捷操作）。
- 多端、同步策略（若未来有手机/Web）。
- 可观测与调试体验：对普通用户或进阶用户的「发生了什么」说明（非仅开发者日志）。
- 引导、空状态、设置项 discoverability。

---

## 2. 架构与工程质量

- **异步与数据**：向量写入幂等键、队列由 JSON 文件演进为 SQLite/专用队列、死信自动重放策略等（参见 [async_outbox.md](async_outbox.md) 中的「最终方案」）。
- **性能**：冷启动、大会话、向量与检索路径优化。
- **稳定性**：更系统的错误边界、重试与降级策略的可视化配置。
- **工程化**：测试覆盖、CI、类型与模块边界、文档与代码同步。

---

## 3. 模型与提示

- Prompt Planner、工具描述、记忆注入策略随模型能力调整。
- 安全与合规（工具边界、外网访问、用户数据提示）。
- 人格、规则、场景块的分层与维护成本（见 [prompt_packaging.md](prompt_packaging.md) 等）。

---

## 4. 运维与数据

- 备份：加密、增量、版本迁移说明、大文件策略。
- 多实例/换机后的**冲突与合并**（若未来需要）。
- 数据目录健康检查、导出/导入的校验与回滚提示。
- **官方 QQ 机器人（合规）**与 Aris 的对接思路见 [qq_bot_official_integration.md](qq_bot_official_integration.md)（非 OneBot）。

---

## 全量导出与迁移

- 备份格式与包含项见 [README 导出/导入](../README.md) 与 [async_outbox.md](async_outbox.md)；后续若在数据目录增加新文件类型，应同步纳入导出/导入逻辑并更新 README「可配置项」或专门说明文档。
