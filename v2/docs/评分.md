这是一个非常关键的问题！**如何让本地模型判断输出结果是否满意**，是整个迭代闭环的核心。这个判断机制设计得好不好，直接决定了系统的效果和效率。

让我从**判断维度、判断方法、评分体系、实践方案**四个层面来回答。

---

## 一、判断维度：从哪些角度评估？

一个完整的满意度判断应该包含以下维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **事实准确性** | 回答是否与记忆/事实一致 | 用户上周说"用新框架"，回答不能写成"用旧框架" |
| **完整性** | 是否覆盖了用户问题的所有方面 | 问"进展和风险"，不能只答进展 |
| **相关性** | 回答是否直接回应问题 | 问项目进展，不能跑题到技术细节 |
| **逻辑性** | 推理链条是否连贯 | 因果关系、时间顺序是否正确 |
| **安全性** | 是否包含敏感/有害内容 | PII、违规内容 |
| **格式正确性** | 输出格式是否符合要求 | 需要JSON时不能给纯文本 |
| **风格匹配** | 是否符合用户偏好 | 用户喜欢简洁，不能啰嗦 |
| **可执行性** | 如果包含指令，是否可执行 | 代码是否能运行、步骤是否清晰 |

## 二、判断方法：本地模型如何做判断？

### 方法1：直接评分法（最简单）

让本地模型直接打分，0-10分或0-100分。

```python
def local_quality_score(response, context, user_input):
    """
    用本地模型直接评分
    """
    prompt = f"""
    请评估以下回答的质量，从0-100分打分。
    
    【用户问题】
    {user_input}
    
    【历史记忆】
    {context}
    
    【回答】
    {response}
    
    【评估维度】
    - 事实准确性：是否与历史记忆一致？
    - 完整性：是否覆盖问题所有要点？
    - 相关性：是否直接回应问题？
    - 逻辑性：推理是否连贯？
    - 安全性：是否包含敏感内容？
    
    请输出格式：
    分数：XX/100
    理由：简要说明扣分原因
    """
    
    result = local_llm.complete(prompt)
    score = extract_score(result)  # 提取分数
    
    return score
```

**优点**：简单直接，一次调用完成  
**缺点**：评分可能不够稳定，不同问题标准不一致

### 方法2：维度拆解法（更稳定）

拆成多个维度分别打分，最后加权平均。

```python
def local_multi_dimension_score(response, context, user_input):
    """
    多维度评分，每个维度独立打分
    """
    dimensions = {
        "factual_accuracy": 0.30,   # 权重30%
        "completeness": 0.20,       # 权重20%
        "relevance": 0.20,          # 权重20%
        "logic": 0.15,              # 权重15%
        "safety": 0.15,             # 权重15%
    }
    
    scores = {}
    
    # 逐个维度判断
    for dim, weight in dimensions.items():
        score = local_dimension_check(dim, response, context, user_input)
        scores[dim] = score
    
    # 加权计算总分
    total_score = sum(scores[dim] * dimensions[dim] for dim in dimensions)
    
    return total_score, scores

def local_dimension_check(dimension, response, context, user_input):
    """
    检查单个维度
    """
    prompts = {
        "factual_accuracy": f"""
            检查回答是否与历史记忆一致。
            历史记忆：{context}
            回答：{response}
            
            如果有不一致的地方，输出0分；完全一致输出100分。
            只输出分数。
        """,
        "completeness": f"""
            用户问题：{user_input}
            回答：{response}
            
            回答是否覆盖了问题的所有方面？
            输出分数（0-100）：
            - 100：完全覆盖
            - 70：覆盖大部分
            - 50：覆盖一半
            - 20：覆盖少部分
            - 0：完全没覆盖
        """,
        "relevance": f"""
            用户问题：{user_input}
            回答：{response}
            
            回答是否直接回应问题？
            输出分数（0-100）
        """,
        "logic": f"""
            回答：{response}
            
            逻辑是否连贯？有无矛盾？
            输出分数（0-100）
        """,
        "safety": f"""
            回答：{response}
            
            是否包含敏感内容（PII、违规内容）？
            包含则输出0分，不包含输出100分。
        """,
    }
    
    result = local_llm.complete(prompts[dimension])
    return int(extract_score(result))
```

**优点**：判断更精准，可解释性强  
**缺点**：多次调用本地模型，延迟增加

让本地模型比较当前输出和理想输出（或标准答案）的差距。

```python
def local_compare_with_standard(response, standard_answer):
```

**适用场景**：有标准答案的场景（如客服对话、教育问答）

### 方法4：规则+模型混合法（推荐实践）

用规则处理确定性判断，用模型处理模糊判断。

```python
def hybrid_quality_check(response, context, user_input):
    """
    规则+模型混合判断
    """
    # 1. 规则检查（快速、确定性）
    rule_score = 100
    
    # 长度检查
    if len(response) < 10:
        rule_score -= 30
    
    # 关键词检查（是否包含"我不知道"等拒答词）
    refusal_keywords = ["我不知道", "无法回答", "不清楚"]
    if any(kw in response for kw in refusal_keywords):
        rule_score -= 50
    
    # PII检查
    if has_pii(response):
        rule_score -= 100  # 直接不合格
    
    # 格式检查
    if expected_format == "json" and not is_valid_json(response):
        rule_score -= 40
    
    # 2. 模型判断（需要理解的部分）
    if rule_score > 50:  # 只有规则通过后才用模型判断
        model_score = local_quality_score(response, context, user_input)
        final_score = (rule_score + model_score) / 2
    else:
        final_score = rule_score
    
    return final_score
```

**优点**：快速、稳定、可解释  
**缺点**：需要维护规则库

## 三、评分体系设计

### 3.1 评分标准

| 分数区间 | 等级 | 含义 | 行动 |
|----------|------|------|------|
| 90-100 | 优秀 | 完美满足需求 | 直接返回 |
| 80-89 | 良好 | 基本满足，有小瑕疵 | 轻微润色后返回 |
| 60-79 | 及格 | 部分满足，有明显问题 | 触发修正 |
| 40-59 | 较差 | 大部分不满足 | 重新生成 |
| 0-39 | 很差 | 完全不合格 | 重新生成+告警 |

### 3.2 阈值设定

```python
class QualityThreshold:
    EXCELLENT = 90   # 直接返回
    GOOD = 80        # 润色后返回
    PASS = 60        # 触发修正
    FAIL = 40        # 重新生成
    
    # 根据场景动态调整
    def get_threshold(self, scenario):
        thresholds = {
            "customer_service": 75,  # 客服场景要求更高
            "creative_writing": 65,  # 创意场景可放宽
            "code_generation": 85,   # 代码场景要求严格
            "casual_chat": 60,       # 闲聊场景可放宽
        }
        return thresholds.get(scenario, self.PASS)
```

```python
def standard_quality_check(response, context, user_input, user_profile):
    """
    标准质量判断
    - 多维度评分
    - 规则+模型混合
    """
    # 1. 快速规则过滤
    rule_violations = check_rules(response)

            "score": 0,
            "satisfied": False,
            "reason": f"规则违反: {rule_violations}",

        "accuracy": check_accuracy(response, context),
        "completeness": check_completeness(response, user_input),
        "relevance": check_relevance(response, user_input),
        "safety": check_safety(response),

    # 4. 决策
    if total_score >= 80:
        action = "return"
    elif total_score >= 60:
        action = "polish"
    else:
        action = "regenerate"
    
    return {
        "score": total_score,
        "dimensions": dimensions,
        "satisfied": total_score >= 60,
        "action": action,
        "feedback": generate_feedback(dimensions)  # 生成修改建议
```

```python
def handle_edge_cases(response, context, user_input):
    """
    处理特殊情况的判断
    """
    # 1. 回答过短
    if len(response) < 20:
        return {"score": 40, "reason": "回答过短", "action": "regenerate"}
    
    # 2. 回答包含拒答词
    refusal_patterns = ["我不知道", "无法回答", "这个问题我无法"]
    if any(p in response for p in refusal_patterns):
        # 检查是否真的无法回答
        if context and len(context) > 0:
            return {"score": 30, "reason": "有记忆但拒答", "action": "regenerate"}
    
    # 3. 回答全是重复内容
    if is_repetitive(response):
        return {"score": 20, "reason": "重复内容", "action": "regenerate"}
    
    return None  # 正常情况
```

### 6.1 持续优化判断模型

```python
class QualityModelOptimizer:
    """
    基于用户反馈优化判断模型
    """
    def __init__(self):
        self.feedback_history = []
    
    def record_feedback(self, user_input, response, auto_score, user_feedback):
        """
        记录用户反馈，用于后续优化
        """
        self.feedback_history.append({
            "input": user_input,
            "response": response,
            "auto_score": auto_score,
            "user_score": user_feedback,  # 用户手动评分
            "timestamp": time.time()
        })
    
    def adjust_thresholds(self):
        """
        根据历史反馈调整阈值
        """
        # 分析自动评分和用户评分的一致性
        inconsistencies = [
            f for f in self.feedback_history 
            if abs(f["auto_score"] - f["user_score"]) > 20
        ]
        
        if len(inconsistencies) > len(self.feedback_history) * 0.3:
            # 一致性低于70%，需要调整
            return {"need_calibration": True}
```
