# 记忆系统优化方案

## 当前问题分析

### 1. 记忆系统性问题
- 记忆检索逻辑分散在多个地方，缺乏统一管理
- 检索策略不够系统化，存在遗忘现象
- 缺乏有效的缓存机制，相同query重复检索

### 2. 工具记忆能力弱
- `read_file`工具每次调用都重新读取文件，无缓存
- 高频读取文件（如`docs/problem_strategy_plan.md`、`memory/aris_ideas.md`）重复读取浪费资源
- 工具利用率低，缺乏智能优化

## 改造方案

### 1. 文件读取缓存系统

#### 核心设计
- 为`read_file`工具添加内存缓存层
- 缓存键：文件路径 + 最后修改时间
- 缓存失效策略：文件修改时间变化时自动失效
- 缓存大小限制：LRU策略，避免内存占用过大

#### 实现要点
```python
class FileCache:
    def __init__(self, max_size=100):
        self.cache = {}
        self.max_size = max_size
        
    def get(self, filepath, mtime):
        key = f"{filepath}:{mtime}"
        return self.cache.get(key)
        
    def set(self, filepath, mtime, content):
        if len(self.cache) >= self.max_size:
            # LRU淘汰
            self._evict_oldest()
        key = f"{filepath}:{mtime}"
        self.cache[key] = {
            'content': content,
            'access_time': time.time()
        }
```

### 2. 记忆检索优化

#### 统一检索管理器
- 集中管理所有记忆检索逻辑
- 实现检索结果缓存，避免重复语义检索
- 智能query生成与优化

#### 检索结果缓存
- 缓存键：query + 上下文摘要
- TTL（Time To Live）机制：短期缓存5分钟，长期重要记忆缓存更久
- 上下文感知的缓存失效

### 3. 工具使用统计与优化

#### 使用模式分析
- 记录工具调用频率、参数模式
- 识别高频读取的文件和常用query
- 分析工具调用时间分布

#### 智能优化策略
- 高频文件预加载：系统启动时预加载常用文档
- 自动建立索引：为常用文档生成关键词索引
- 摘要生成：对长文档生成简短摘要，减少重复读取

### 4. 系统化记忆架构

#### 三层记忆架构
```
┌─────────────────┐
│   短期缓存层    │
│  - 工具结果缓存 │
│  - 文件内容缓存 │
│  - 检索结果缓存 │
└─────────────────┘
         ↓
┌─────────────────┐
│   中期记忆层    │
│  - 对话上下文   │
│  - 用户近期偏好 │
│  - 会话状态     │
└─────────────────┘
         ↓
┌─────────────────┐
│   长期记忆层    │
│  - 用户身份     │
│  - 重要要求     │
│  - 核心关联     │
└─────────────────┘
```

#### 各层特点
- **短期缓存层**：快速访问，自动失效，内存驻留
- **中期记忆层**：会话级持久化，上下文相关
- **长期记忆层**：永久存储，核心用户认知
